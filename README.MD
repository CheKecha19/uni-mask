# Data Masker

Универсальный скрипт для маскирования и восстановления чувствительных данных в текстовых файлах.  
Подходит для быстрого обезличивания логов, конфигураций, отчётов перед передачей третьим лицам.

## Возможности

Заменяет следующие типы данных на псевдонимы (с сохранением маппинга):

- **IP-адреса** (IPv4)
- **Слово "sberbank"** (в любом регистре)
- **Хеши** MD5, SHA-1, SHA-224, SHA-256, SHA-384, SHA-512
- **Email-адреса**
- **Номера телефонов** (российские и международные)
- **Номера кредитных карт** (с проверкой по алгоритму Луна)
- **СНИЛС** (РФ)
- **ИНН** (РФ)
- **Паспортные данные РФ** (серия + номер)

Все замены уникальны: одинаковые исходные значения получают одинаковые псевдонимы.  
Маппинг сохраняется в JSON-файл, что позволяет полностью восстановить исходные данные.

## Установка

Скрипт написан на Python 3, не требует сторонних библиотек.  
Скачайте файл `masker.py` и запускайте из командной строки.

```bash
python masker.py --help
```

## Использование

### Маскирование

```bash
python masker.py --mode mask --input file.txt --output masked.txt --mask map.json
```

Если не указать `--output` и `--mask`, они создаются автоматически:
- `file.masked.txt`
- `file.mask.json`

### Восстановление

```bash
python masker.py --mode unmask --input masked.txt --output restored.txt --mask map.json
```

При восстановлении файл маски обязателен.

### Пользовательские префиксы

Для каждой категории можно задать свой префикс через аргументы вида `--<категория>-prefix`:

```bash
python masker.py --mode mask -i secret.txt --ip-prefix IP --email-prefix EM --credit-card-prefix CARD
```

Список категорий и префиксов по умолчанию:

| Категория      | Префикс (по умолчанию) | Аргумент для смены       |
|----------------|------------------------|--------------------------|
| IP             | ip                     | `--ip-prefix`            |
| sberbank       | sbr                    | `--sberbank-prefix`      |
| хеши           | hash                   | `--hash-prefix`          |
| email          | email                  | `--email-prefix`         |
| телефон        | phone                  | `--phone-prefix`         |
| кредитная карта| cc                     | `--credit-card-prefix`   |
| СНИЛС          | snils                  | `--snils-prefix`         |
| ИНН            | inn                    | `--inn-prefix`           |
| паспорт РФ     | passport               | `--passport-rf-prefix`   |

## Пример

Исходный файл `data.txt`:

```
Сервер 192.168.1.10, email user@example.com, телефон +7 (495) 123-45-67.
Карта 4111 1111 1111 1111, снилс 123-456-789 00.
```

Маскирование:

```bash
python masker.py -m mask -i data.txt
```

Результат `data.masked.txt`:

```
Сервер ip1, email email1, телефон phone1.
Карта cc1, снилс snils1.
```

Восстановление:

```bash
python masker.py -m unmask -i data.masked.txt -k data.mask.json
```

Получим исходный файл.

## Примечания

- Для номеров кредитных карт выполняется проверка по алгоритму Луна, что уменьшает ложные срабатывания.
- Телефонные номера определяются по упрощённому регулярному выражению – при необходимости можно доработать.
- Все замены сохраняют исходное написание (например, "Sberbank" заменится на `sbr1`, но при восстановлении вернётся именно "Sberbank").
- При большом количестве уникальных значений файл маски может стать объёмным – это нормально.

## Расширение списка типов данных

Чтобы добавить новый тип, отредактируйте словарь `PATTERNS` в начале скрипта. Формат записи:

```python
'название': (r'регулярное_выражение', 'префикс_по_умолчанию', функция_валидации_или_None)
```

Функция валидации принимает строку и возвращает `True`, если это действительно искомый тип (используется для дополнительной фильтрации).
